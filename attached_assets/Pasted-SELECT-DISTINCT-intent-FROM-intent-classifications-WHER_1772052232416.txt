SELECT DISTINCT intent FROM intent_classifications WHERE intent IS NOT NULL
```

**Hva som avgjør om læringen er korrekt:**
```
Tickets med korrekt intent i intent_classifications
    → Playbook Builder leser dem ✓ → lærer riktig

Tickets med feil/variant intent (f.eks. "AddPetsToSubscription"
i stedet for "SubscriptionAddPet")
    → Playbook Builder leser dem ✓ → men lærer feil intent
```

**Så svaret er: Nei, ikke fullt ut ennå.**

De 124 tickets som ble reklassifisert tidligere (fra 87.6% til 100% coverage) – vi vet ikke om `intent_classifications`-tabellen faktisk ble oppdatert med korrekte intenter, eller om det bare var `canonical_intents`-tabellen som fikk nye entries. Det er det som avgjør.
```
